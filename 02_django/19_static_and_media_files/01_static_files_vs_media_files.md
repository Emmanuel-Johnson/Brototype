## Django Settings Comparison

```python

```

```python

```

---

## How Django Treats Them

### Static Files

- Collected using `collectstatic`
- Served by a web server in production
- Not uploaded via forms

---

### Media Files

- Saved to `MEDIA_ROOT`
- Uploaded via forms
- Served by web server or cloud storage

---

## Production Rule (Very Important)

âŒ Django should **NOT** serve static or media files in production  
âœ… Use **Nginx / Apache / S3 / CDN**

## Interview One-Liner

**Static files are developer-owned assets; media files are user-uploaded content.**

## Quick Summary

- Static = CSS, JS, logos  
- Media = uploads, images, files  
- Static is fixed, media is dynamic  
- Django treats them separately

---

### Minimum Recommended Setup

```python

    BASE_DIR / "static"
]

STATIC_ROOT = BASE_DIR / "staticfiles"
```

---

### Why These Settings Exist

- **STATIC_URL**  
  URL prefix used in templates  
  Example: `/static/style.css`

- **STATICFILES_DIRS**  
  Where you keep static files during **development**

- **STATIC_ROOT**  
  Where Django collects **all static files** when you run:

```bash
python manage.py collectstatic
```

âš ï¸ `STATIC_ROOT` is **mandatory in production**.
